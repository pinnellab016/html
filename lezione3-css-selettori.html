1. Perche usare CSS

HTML descrive la struttura, CSS descrive l'aspetto. Separarli rende il lavoro piu ordinato e piu veloce da correggere.

Prova subito

HTML struttura

Titoli, paragrafi, link, sezioni e contenuti. Senza CSS la pagina resta funzionale, ma poco curata.

<h1>Titolo</h1> <p class="evidenziato">Testo importante</p>

CSS aspetto

Colori, sfondi, spaziature, dimensioni del testo e peso del font. Un file puo controllare molte pagine.

.evidenziato { color: red; background-color: yellow; }

2. Sintassi e tipi di selettore

Un selettore sceglie quali elementi HTML riceveranno le proprieta CSS.

h1 selettore di tag

Colpisce tutti gli elementi di quel tipo. Utile per gli stili globali.

h1 { color: blue; }

.classe selettore di classe

Colpisce tutti gli elementi con quella classe. E ripetibile.

.evidenziato { color: red; }

#id selettore di ID

Colpisce un elemento specifico. L'ID deve essere unico nella pagina.

#titolo-principale { color: green; }
Selettore Quando usarlo Esempio HTML
p, h1, body Per stili generali su tutti gli elementi dello stesso tipo. <p>Testo</p>
.evidenziato Per uno stile riutilizzabile su piu elementi. <p class="evidenziato">...</p>
#titolo-principale Per un elemento unico e riconoscibile. <h1 id="titolo-principale">...</h1>

Regola rapida: nel CSS la classe vuole il punto, l'ID vuole il cancelletto. In HTML invece si scrive class="..." oppure id="...".

Errori comuni

            
    
              
  • Dimenticare le virgolette in class="evidenziato".
    • 
              
  • Scrivere evidenziato invece di .evidenziato nel CSS.
    • 
              
  • Scrivere titolo invece di #titolo nel CSS.
    • 
              
  • Mettere il file CSS in una cartella diversa senza correggere href.

  • Usare lo stesso ID su piu elementi.
